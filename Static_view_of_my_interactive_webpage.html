Data Visualization Assignment : Static view of our Interactive webpage

This Python web application can be used to optimize a portfolio of assets and display interactively the results. To get interactive plots the user should run our python scripts. But Installing python libraries can be difficult, to alleviate this problem we created a Software (binary file or .Exe) that runs on Windows MIPOpti. We used two visualization libraries plotly and dash, and one optimization library cvxopt. This webpage was entirely created using Dash, no html template was used. We downloaded the returns data of 10 stocks from the website of the University of Pennsylvania.

Encountered difficulties:

  • Mastering Dash and plotly to produce high quality graphs
    • handling the properties of plot objects to render nice figures
    • Advanced concepts like "Callbacks" : callbacks enable the update of figures as the inputs change
    • Updating multiple figures by creating multiple callbacks
    • Writing html instructions from Dash

Historical returns of a basket of assets

Interpretation : The monthly returns of the assets show their performances. Assets with high variations can yield higher returns, but they are more volatile and therefore riskier.

Linear relationships between assets returns

Interpretation : The correlation matrix highlights the correlations (linear relationships) between the returns of all pair of assets in the portfolio. The higher the correlation the more the assets change (increase or decrease) in the same way. A good portfolio should contain less correlated assets (close to zero or less than one). The correlation matrix is used as the main input in the optimization of a portfolio.

Solution of the optimization of a portfolio

Interpretation : The efficient frontier (blue discrete points on the figure) is the solution of the optimization of a portfolio. For each risk level the higher return is a point that belongs to the efficient frontier. The capital allocation line can be used to determine how to share a bugdet between risky and a risk-free asset.

Budget Allocation between risky and risk-free assets

Interpretation : The allocation of a bugdet of 10k euros between risky and one risk free asset. Assets with good risk-return (i.e. BR, CDN, BF, risk-free) trade-offs have positive allocation.
